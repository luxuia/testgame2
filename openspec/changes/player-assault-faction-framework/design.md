## Context

设定总纲要求“其他阵营角色从固定位置出生，并依据目标规划做事情”，且当前仓库已具备 GOAP-Lite、CombatActionPipeline、CrowdAgentController 与主客场权限网关。现阶段缺口不是底层能力，而是阵营级编排：谁在何处出生、何时进攻、如何把阵营目标稳定映射到单位行为循环。

## Goals / Non-Goals

**Goals:**
- 在现有系统上新增阵营编排层，支持固定出生点驱动的敌对阵营生成。
- 建立首版“进攻玩家阵营”目标循环：集结、推进、攻击、重组。
- 复用现有 GOAP 与 Combat 共享执行管线，避免双轨逻辑。
- 确保地形相关动作继续受主客场权限网关约束。
- 提供低成本、可验证、可扩展到多阵营的骨架。

**Non-Goals:**
- 不实现多阵营外交、关系演化与交易系统。
- 不实现复杂剧情导演或事件叙事系统。
- 不重写寻路、区块、渲染与动画底层。
- 不在首版中做阵营平衡与内容量扩展（仅攻方 MVP）。

## Decisions

1. 新增阵营主控 `FactionScenarioDirector` 负责场景级编排  
- 选择：以单一主控管理阵营运行状态、波次触发、单位生命周期。  
- 原因：跨模块依赖最少，便于在 `SinglePlayer` 直接挂载验证。  
- 备选：把逻辑分散到各单位控制器。被拒绝，因为会导致波次与全局目标难以统一。

2. 出生系统采用“固定锚点 + 批次生成”  
- 选择：每个阵营使用一组 `SpawnAnchor`（场景 Transform）作为出生位置来源。  
- 原因：满足“固定位置出生”的设定要求，并便于关卡调试与复盘。  
- 备选：随机半径刷怪。被拒绝，因为不满足设定且可重复性差。

3. 阵营行为使用“阶段机 + 目标权重”而不是复杂行为树  
- 选择：阶段定义为 `Assemble -> Advance -> Attack -> Regroup`，阶段内由目标权重驱动。  
- 原因：低成本且可解释，能快速接入现有 GOAP-Lite。  
- 备选：全量行为树/叙事导演。被拒绝，因为超出首版成本边界。

4. 单位决策通过桥接器复用现有 GOAP 与 Combat 管线  
- 选择：`FactionAgentBrainBridge` 将阵营级目标映射为 GOAP 黑板与动作过滤，再走 `CombatActionPipeline` 执行。  
- 原因：保留现有 legality、失败原因与回退一致性。  
- 备选：阵营系统直接调用移动/攻击接口。被拒绝，因为会绕开权限与失败处理合同。

5. 首版目标对象优先级固定为“玩家 > 菌核核心（可配置权重）”  
- 选择：默认优先攻击玩家实体，不可达时切换菌核核心。  
- 原因：符合“进攻玩家阵营”定义，同时保留客场压力。  
- 备选：始终攻击菌核。被拒绝，因为与用户提出的首版目标不一致。

6. 地形动作维持共享权限网关，不引入阵营特权  
- 选择：敌方单位地形相关动作仍必须经过 Home/Away `CanEditTerrain()` 约束。  
- 原因：避免规则分叉，保持与总纲“菌毯即权柄”一致。  
- 备选：敌方在 Away 无限制编辑。被拒绝，因为会破坏既有规则与平衡。

7. 阵营配置采用模板资产，不新增顶层全局配置键  
- 选择：使用 `FactionDefinition` / `FactionObjectivePreset` / `FactionSpawnPreset`（或等价结构）承载参数。  
- 原因：与现有“最小全局键”策略一致，降低配置爆炸风险。  
- 备选：新增全局运行时键。被拒绝，因为首版不需要全局扩键。

## Risks / Trade-offs

- [风险] 固定锚点密度不足导致出生拥堵或卡位 -> 缓解：加入锚点占用检测与备用锚点回退。
- [风险] 阶段切换过频引发行为抖动 -> 缓解：为阶段设置最短驻留时间与切换冷却。
- [风险] 目标优先级固定可能导致战术单一 -> 缓解：通过目标权重模板保留后续扩展口。
- [风险] Away 权限限制使敌方地形战术偏弱 -> 缓解：首版仅启用进攻闭环，后续版本通过可控临时权限扩展。
- [风险] 阵营主控成为集中热点 -> 缓解：限制每帧处理预算，单位执行保持分帧。

## Migration Plan

1. 在 OpenSpec 固化 proposal/design/spec/tasks，冻结首版边界。  
2. 实现阶段先落 `FactionScenarioDirector` 与固定出生点数据合同。  
3. 接入 `FactionAgentBrainBridge`，打通到 GOAP 与 Combat 共享管线。  
4. 在 `SinglePlayer` 验证进攻循环（集结/推进/攻击/重组）与权限约束。  
5. 若首版不稳定，回滚到“仅固定出生 + 直线推进 + 基础攻击”保底模式。

## Open Questions

- 首版是否需要“夜晚加压/白天收缩”的阵营节奏，还是保持常压进攻。
- 玩家与菌核目标切换的距离阈值是否按阵营模板可调。
- 敌方在 Away 场景是否允许极小半径临时编辑权限（首版默认不启用）。
