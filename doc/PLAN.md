### 1.4 核心调整总纲（适配独立开发，大幅降低难度，贴合环世界多工人协作玩法）

#### 1.4.1 调整核心原则

1.  玩法适配原则：彻底转向环世界核心玩法，聚焦“多工人菌菇协作+生存管理+资源采集+基地建设”，摒弃原有菌主相关设定，仅保留多个工人菌菇形象及可爱设定作为核心角色；

2.  难度适配原则：彻底简化系统，摒弃复杂战斗、多结局、权柄联动等难开发内容，贴合环世界轻量生存建设+多工人管理逻辑，适配独立开发能力；

3.  低成本原则：优先复用免费素材、开源插件，简化美术制作、代码开发，避免高难度算法、复杂逻辑，控制开发工作量；

4.  简易操作原则：俯视角操作简化，核心操作仅保留“建筑建造、资源收集、工人菌菇任务分配与状态管理”，方块破坏/放置由工人菌菇执行（玩家分配对应任务），降低玩家学习成本，也减少开发难度；

5.  聚焦核心原则：放弃冗余内容，仅保留环世界类“多工人协作生存城镇建设”核心玩法，确保开发周期可控、功能可落地，避免贪多导致无法完成。

#### 1.4.2 核心保留与简化内容

1.  保留内容：我的世界体素美术风格（复用基础体素素材）、俯视角视角、方块自由改造（由工人菌菇执行，简化操作逻辑）、环世界类生存城镇建设核心（生存管理、资源采集、基地建设）、多个工人菌菇形象及可爱设定（作为核心可协作角色，无原有菌主设定）；

2.  简化内容：生存系统（仅保留饥饿、疲劳、健康3种核心状态，简化数值逻辑，适配多工人管理）、资源系统（仅保留3-4种核心资源，简化获取/管理）、建筑系统（简化建造条件，减少建筑种类，贴合基地生存需求）、工人管理（简化多工人分工，仅保留基础采集、建造分工，适配独立开发）；

3.  删除内容：原有菌主所有设定（菌毯机制、菌主权柄等）、复杂战斗系统、七罪权柄体系、多副官托管、多剧情分支、多平台适配，彻底降低开发难度，聚焦环世界多工人协作核心玩法。

## 3. 验收总原则与优先级定义（简化，适配独立开发）

### 3.1 验收核心准则（简化，务实落地）

1.  可落地原则：所有交付物以“能运行、能使用”为核心，不追求完美，优先保障核心功能落地；

2.  灵活调整原则：可根据开发难度，灵活调整排期与功能细节，优先完成核心玩法，冗余功能可删减；

3.  低成本原则：验收不追求复杂效果，美术、功能达到基础可用标准即可，降低独立开发压力；

4.  玩法贴合原则：核心玩法需贴合环世界类“多工人协作生存建设”逻辑，多个工人菌菇作为核心可协作角色，形象贴合可爱设定，无原有菌主设定。

### 3.2 需求优先级定义（简化，聚焦核心）

|优先级|定义|交付要求|
|---|---|---|
|P0|核心必做，游戏能正常运行、环世界类多工人协作核心玩法可体验|必须按时交付，不允许延期，是项目落地的基础|
|P1|重要补充，提升体验但非必备|可灵活调整，核心功能完成后再补充|
|P2|优化内容，可有可无|时间充足则做，不足可直接放弃|
