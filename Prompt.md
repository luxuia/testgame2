# Prompt.md

## 1) 目标与非目标

### 目标
- 稳定单角色与多人（300+）自动寻路流程，避免卡住、重复重算、无效更新。
- 在现有框架内持续优化，不推翻已有 A* + FlowField 体系。
- 建立可验证的性能观测能力：每次寻路可看到耗时和遍历节点量。
- 保证点击选中、移动、攻击、绕障路径在常见地形下可用。

### 非目标
- 不在本阶段重写为 ECS / DOTS / Job System 全量方案。
- 不在本阶段替换世界存储结构或区块加载架构。
- 不做与寻路无关的 UI/美术/音频改造。
- 不做跨平台适配以外的引擎大版本升级。

## 2) 硬性约束

- 性能约束：
- `TargetSelector.Update` 与 `CrowdFlowFieldCoordinator.Update` 禁止出现明显每帧高频 GC 分配热点。
- 单角色点击寻路不应触发大规模无意义 FlowField 构建。
- 多角色同目标场景应优先复用场流缓存，避免每个 agent 独立 A*。

- 确定性约束：
- 同一世界状态 + 同一起终点，寻路结果应稳定（允许等价路径轻微差异，但不允许明显随机抖动）。
- 失败重试应有冷却，避免每帧重复失败计算。

- UX 约束：
- 点击后不应回弹到格子中心。
- 需要绕障、跳高差时路径应可持续推进，卡住时应有重寻路恢复。
- 进入攻击范围后应可正常攻击目标方块。

- 平台约束：
- Unity 工程 `UnityProject` 与 `Assembly-CSharp.csproj` 可编译通过。
- Windows 开发环境可直接运行验证命令。

## 3) 交付物

- 代码交付：
- 稳定的单角色路径选择与移动控制流程。
- 面向人群寻路的 `CrowdFlowFieldCoordinator` 协调流程。
- 可开关的寻路性能日志（耗时、节点量、算法类型、缓存命中信息）。

- 文档交付：
- `Prompt.md`（本文件）
- `Plan.md`
- `Implement.md`
- `Documentation.md`

## 4) Done when（验收标准）

- 编译通过：
- 运行 `dotnet build UnityProject/Assembly-CSharp.csproj -nologo`，结果为 `0 errors`。

- 行为正确：
- 单击目标后角色可移动到可达站位；遇到障碍可绕行；需要跳跃时可继续推进。
- 在攻击范围触发后可持续攻击并完成方块破坏。

- 性能可观测：
- 每次寻路都能输出结构化日志，包含 `timeMs` 和 `expandedNodes/buildExpanded`。
- 可按开关关闭日志，关闭后不产生该类调试输出。

- 范围受控：
- 本轮仅修改寻路相关代码和文档，不扩展到无关模块。
